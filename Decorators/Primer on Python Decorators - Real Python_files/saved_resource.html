    Just as a side note i use inner functions like this.

    def env():

    env.add = add

                                                  Hi guys. First I’d like to thank you for the great article. However I got stuck and I need your help in understanding one thing related to this part: just_some_function = my_decorator(just_some_function).

                                                  Let’s focus on your Example 1. You first define my_decorator function() and then you define just_some_function(). All clear here. If you type just_some_function() in the terminal you will get Wheee! in output as expected. But then you do the following: just_some_function = my_decorator(just_some_function). I explained that to myself like this: result of function my_decorator, called with function just_some_function as argument, is assigned to just_some_function. The result is function wrapper. All clear here too. But, it happens that just_some_function is the name of the function that was used as the argument in my_decorator. The end result is that original just_some_function() is not functional anymore, that is, it appears as if it was re-defined. If you now type just_some_function() in the terminal you will NOT get Wheee! (as expected). You will get decorated version of that function. Which is cool but the original definition of the just_some_function() is now lost and cannot be used from this point on. So my question is what would be the motivation for permanently modifying (decorating) the original function?
